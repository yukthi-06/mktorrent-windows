# Initial Analysis
The build issues are caused by:
1. `b - a` has type `long long int` (or `ptrdiff_t` on 64-bit Windows), which mismatching `%lu`.
2. Standard BSD/POSIX random functions `random()` and `srandom()` are missing under MinGW-w64 on Windows. They should be mapped to the standard C `rand()` and `srand()`.

# Assumptions
- Typecasting pointer difference to `(unsigned long)` is safe and standard since the maximum directory entry/filename length will easily fit in 32 bits.
- Using `rand()` and `srand()` as replacements for `random()` and `srandom()` respectively on Windows is sufficient and correct.

# Risks
- `rand()` typically has a lower period or random quality compared to POSIX `random()`, but it is completely standard and sufficient for seeding the cross-seed random data (which is just some hex characters).
