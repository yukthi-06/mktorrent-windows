# Build Windows 64-bit EXE on GitHub Actions

This implementation plan details the addition of a GitHub Actions workflow that compiles and builds the `mktorrent` C codebase into a Windows 64-bit executable.

## User Review Required

> [!IMPORTANT]
> The GitHub Actions workflow will compile `mktorrent` with `USE_PTHREADS=1` to enable multi-threaded hashing support (which is standard and recommended for performance on Windows).
> We will also produce the executable using the standard `mingw-w64` GCC toolchain on an `ubuntu-latest` runner (cross-compilation). This is extremely fast, highly reproducible, and lightweight.

## Proposed Changes

### GitHub Actions Configuration

#### [NEW] [build-windows.yml](file:///x:/Projects_X/0_Active/00/mktorrent/.github/workflows/build-windows.yml)
Create a new workflow file at `.github/workflows/build-windows.yml` to compile the project.

```yaml
name: Build Windows 64-bit EXE

on:
  push:
    branches: [ "*" ]
  pull_request:
    branches: [ "*" ]
  workflow_dispatch:

jobs:
  build:
    name: Build Windows 64-bit EXE
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Install MinGW Toolchain
        run: |
          sudo apt-get update
          sudo apt-get install -y mingw-w64 make

      - name: Compile mktorrent
        run: |
          make CC=x86_64-w64-mingw32-gcc USE_PTHREADS=1

      - name: Rename executable
        run: |
          mv mktorrent mktorrent.exe

      - name: Upload Build Artifact
        uses: actions/upload-artifact@v4
        with:
          name: mktorrent-windows-x64
          path: mktorrent.exe
```
