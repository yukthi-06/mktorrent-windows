# User Prompt
```
torrent file is created (with zero size) and program is stuck for a long time.
Had to Control+C to kill it
```

# Initial Analysis
The program hangs during the hashing phase inside `make_hash()`.
In `hash_pthreads.c`, the main thread creates a progress-printing thread running `print_progress()` and subsequently calls `pthread_cancel(print_progress_thread)` followed by `pthread_join(print_progress_thread)`.
On Windows, `winpthreads` does not reliably implement asynchronous thread cancellation (`pthread_cancel`). The thread cancellation either behaves as a no-op or causes the calling thread to deadlock/hang indefinitely when joining the progress thread.
To resolve this, we will replace `pthread_cancel` with a thread-safe flag (`stop_progress`) in the queue structure. The progress thread will periodically poll this flag and exit cleanly, avoiding any call to `pthread_cancel`.

# Assumptions
- The hang is caused by `pthread_cancel` failing to terminate the progress thread, leading to an infinite block on `pthread_join`.

# Risks
- None. Polling a flag is a standard, robust, and portable way of ending threads.
