# Initial Analysis
When compiling with `x86_64-w64-mingw32-gcc` and `-lpthread`, the resulting binary dynamically links against MinGW runtime libraries such as `libwinpthread-1.dll` and `libgcc_s_seh-1.dll`.
If these DLLs are not in the Windows path, the executable exits silently (or throws a DLL not found error) before `main()` even runs.
To make the executable fully standalone and portable, we must link it statically using the `-static` flag.

# Assumptions
- The silent exit is caused by missing dynamic runtime DLLs (`libwinpthread-1.dll` / `libgcc_s_seh-1.dll`).
- Linking statically (`-static`) will embed these runtimes into `mktorrent.exe`.

# Risks
- Static linking slightly increases the binary size, but this is negligible and standard practice for standalone Windows executables.
